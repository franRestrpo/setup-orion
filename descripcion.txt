El proyecto Setup-Orion automatiza la instalación y configuración de la plataforma Orion en un entorno Docker Swarm utilizando Ansible.

**Arquitectura:**

El sistema se gestiona a través de un script de Python (`cli/setup_orion.py`) que actúa como un lanzador para los playbooks de Ansible. Toda la lógica de aprovisionamiento, configuración y despliegue está contenida dentro de la estructura de Ansible para asegurar la idempotencia y la modularidad.

**Flujo de Ejecución:**

1.  **`cli/setup_orion.py`**: Es el punto de entrada. Pregunta al usuario si desea instalar o desinstalar la plataforma.
2.  **`cli/ansible/playbooks/site.yml`**: Este es el playbook principal para la instalación. Orquesta una serie de roles en el siguiente orden:
    *   `prechecks`: Verifica que el sistema operativo y la arquitectura sean compatibles.
    *   `base`: Actualiza el sistema, configura la zona horaria (`America/Bogota` por defecto) e instala paquetes base necesarios (`git`, `swaks`, `apache2-utils`, `neofetch`, etc.).
    *   `docker`: Instala y configura Docker Engine.
    *   `swarm`: Inicializa el modo Docker Swarm si no está activo y crea la red overlay `DOCKER-SWARM` para la comunicación interna de los servicios.
    *   `orion`: Despliega el stack de la aplicación Orion usando `docker-compose.yml`.
3.  **`cli/ansible/playbooks/remove.yml`**: Este playbook se encarga de la desinstalación, eliminando el stack de Orion del clúster Swarm.

**Uso:**

Para ejecutar el instalador, simplemente corra el script de Python y siga las instrucciones:
```bash
python3 cli/setup_orion.py
